the history reads like real incremental work rather than a single dump.

## Why this structure

Bundling by topic arc (rather than one PR per lab unit, or one PR for everything)
mirrors how the course itself built skills in stages: data intake, then
visualization, then cleaning/wrangling, then aggregation/time series, then the two
capstones. It also keeps each PR's diff small enough to actually review.

## Reproducing the build artifacts

Every chart image and every notebook PDF in this repo was generated by:

```bash
python scripts/build_artifacts.py    # extracts PNGs to assets/images/, renders PDFs
python scripts/generate_readme.py    # regenerates README.md from assets/images/
```

Re-run both after editing any notebook to keep the gallery and PDFs in sync.
